# Применение нейросетей для статистической обработки информации

## Краткое описание

Репозиторий содержит воспроизводимую цепочку подготовки, проверки качества и первичного анализа неоднородного набора данных. Прикладной набор связан с наблюдениями за изменением береговой бровки Волгоградского водохранилища, однако основной предмет работы здесь не сама экологическая интерпретация, а методы статистической и нейросетевой обработки информации.

На текущем этапе проект показывает, как из разнородных исходных таблиц формируется аккуратный производный слой, пригодный для описательного анализа, интервальных расчётов и сравнения профилей внутри участков. В базовое сравнение добавлен первый нейросетевой ориентир `MLPRegressor`, а отдельный демонстрационный автоэнкодер используется для поиска подозрительных наблюдений.

## Что уже сделано

- разобраны и нормализованы исходные таблицы по участкам, профилям, береговым наблюдениям, ветру, воде и базисным точкам;
- сформирован воспроизводимый производный слой с полями происхождения данных и контекстом проверки качества;
- рассчитаны интервальные метрики изменения береговой бровки;
- подготовлен безопасный поднабор для первого аналитического этапа;
- построены графики по периодам наблюдений, интенсивности изменений и межпрофильным корреляциям;
- добавлено базовое моделирование на производном датасете, включая первый нейросетевой ориентир `MLPRegressor`;
- добавлен демонстрационный нейросетевой поиск подозрительных наблюдений через автоэнкодер;
- добавлены QC-флаги и текстовые ограничения интерпретации.

## Что пока не является финальным выводом

- данные по ветру требуют дальнейшей проверки полноты и покрытия на уровне интервалов;
- уровни воды пока используются как ограниченный годовой контекст и требуют более аккуратной расшифровки для объяснительного анализа;
- конфликтующие дубли в исходных береговых наблюдениях не скрываются и требуют предметного решения;
- знак смещения в профильной системе координат нельзя автоматически трактовать как готовый физический вывод без доменной проверки;
- базовое моделирование даёт первичную оценку предсказательного сигнала, но кросс-валидация и групповая проверка могут давать другую картину;
- `MLPRegressor` и автоэнкодер используются как первые нейросетевые демонстрационные ориентиры; они не заменяют экспертную проверку и не считаются финально настроенными моделями;
- строки с высокой ошибкой восстановления автоэнкодера не удаляются автоматически и не считаются доказанными ошибками данных.

## Структура репозитория

- `data/raw/` — исходные файлы, которые не редактируются.
- `data/interim/` — промежуточные QC-артефакты, словари, логи и review-таблицы.
- `data/processed/` — основные производные таблицы для анализа и показа.
- `notebooks/` — поясняющие notebook первого этапа анализа.
- `src/` — парсеры, сборка датасетов, QC и аналитические сценарии.
- `reports/figures/` — готовые графики первого этапа.
- `reports/tables/` — сводные таблицы, словари и QC-отчёты.
- `archive/` — архивные материалы, не входящие в основной воспроизводимый контур.

## Как пересобрать проект

Установка зависимостей:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -e ".[dev]"
```

Или через `make`:

```bash
make install
```

Основные команды проекта:

```bash
make build-datasets
python -m src.pipeline
python -m src.analysis.first_stage_analysis
python -m src.analysis.baseline_modeling
python -m src.analysis.autoencoder_anomaly_detection
make qc
pytest
```
